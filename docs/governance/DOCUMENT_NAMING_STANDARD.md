# Document Naming Standard

## Naming Pattern
Documents should follow this convention where possible:
`DOMAIN_TOPIC_DOCUMENT-TYPE[_VERSION].ext`

## Examples
- `FE_ROUTE_META_CONTRACT.md`
- `FE_DESIGN_SYSTEM_GUIDE.md`
- `BE_AUTH_ENDPOINT_INVENTORY.csv`
- `BE_APPLICATION_WORKFLOW.md`
- `DB_DATA_DICTIONARY.xlsx`
- `UX_TEACHER_ATTENDANCE_FLOW.md`
- `INT_FE_BE_CAPABILITY_MATRIX.csv`
- `TEST_BROWSER_SMOKE_REPORT.md`
- `P0_IMPLEMENTATION_REPORT.md`

## Principles
1. Preserve recognizable names but place them in the correct domain folders.
2. Only rename vague files (e.g. avoid `final.md`, `final2.md`, `new.md`, `copy.md`).
3. Update all references if a rename is necessary.
